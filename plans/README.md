## Direction findings

- Durable board artifact: The project vocabulary now includes Debrief Board, Coaching Tile, Tile Source Snapshot, Tile Display Text, and Edited Coaching Tile. That vocabulary is wasted if edited boards cannot be saved as portable local files.
- Tile preset editor, not freeform canvas: `CONTEXT.md` rejects "freeform layout" language for Debrief Layout Presets. Improve structure editing within presets before considering drag/drop windows.
- Source-aware formatter tests: This static HTML file has enough domain logic that a small `?selftest=1` harness is justified. It should not grow into a build pipeline unless the app moves out of standalone docs.

## Findings considered and rejected

- Replace the static HTML file with React/Vite: rejected for now. The repo's current docs artifact pattern is no-build static HTML, and the requested improvement is local editing behavior, not an app-stack migration.
- Add drag-and-drop resizing: rejected for now. It conflicts with the repo vocabulary that Debrief Layout Presets constrain composition and avoid a generic canvas.
- Change deterministic analysis or selected-delta schema: rejected. The editing feature gaps are in `docs/crt_terminal_text.html`; deterministic analysis owns statuses, evidence, and Reportable Delta selection.
